# 回归分析项目展示报告
## 一、Task1 实现方式说明（Class 面向对象）
本次作业在 Task1 中选择了**面向对象（Class）**的方式实现 `CustomOLS` 回归引擎，而非过程式函数写法。

### 1. 实现思路
将线性回归所需的核心方法 `fit()`、`predict()`、`score()`、`f_test()` 封装在一个类中，同时将训练得到的系数、残差方差、协方差矩阵、自由度等结果统一存储在对象内部（`self` 属性中），实现数据与行为的完整封装。

### 2. 在多市场建模中的优势
在 Task3 同时对北美（NA）和欧洲（EU）两个市场建模时，面向对象的优势非常明显：
1. **多实例完全隔离**  
   可以直接创建两个独立对象 `model_na` 和 `model_eu`，两个模型的系数、检验结果互不干扰、互不覆盖，不会出现变量混淆或参数传错的问题。
2. **代码结构更简洁**  
   不需要在函数之间反复传递 `beta_hat`、`cov_matrix`、`sigma2` 等大量中间变量。
3. **符合工业级接口规范**  
   统一的 API 设计可以直接与 `sklearn` 模型共用同一个评估函数，体现了鸭子类型（Duck Typing）的设计思想。
4. **扩展性更强**  
   后续新增模型、新增检验方法时，只需在类内部扩展，不影响整体代码结构。

### 3. 缺点
- 代码结构比单纯的过程式函数稍复杂；
- 对初学者理解成本略高；
- 在单模型场景下优势不明显。

## 二、北美与欧洲市场广告效果差异解读
### 1. F 检验结果说明
本次对两个市场均执行了**广告渠道联合显著性 F 检验**，原假设为：
**电视、广播、社交媒体三类广告的系数全部为 0，即广告投放对销量无显著影响**。

结果如下：
- 北美市场：F统计量 = 54450.3336，P值 = 0.0000 < 0.05 → **显著拒绝原假设**
- 欧洲市场：F统计量 = 68786.6881，P值 = 0.0000 < 0.05 → **显著拒绝原假设**

**大白话解释**：
无论是北美市场还是欧洲市场，广告整体投放策略都**非常有效**，广告投入确实能显著带动销量提升，不是随机波动带来的结果。

### 2. 具体广告效果差异
模型系数顺序：`[截距, TV_Budget, Radio_Budget, SocialMedia_Budget, Is_Holiday]`

#### （1）北美市场（NA）
- TV 广告效果强：系数 3.5075
- 广播广告效果强：系数 3.4977
- 社交媒体广告几乎无效：系数 0.0021
- 节假日对销量提升非常明显：系数 26.699

**结论**：北美市场更依赖传统媒体广告（电视+广播），社交媒体投放性价比极低，节假日效应显著。

#### （2）欧洲市场（EU）
- 广播广告效果最强：系数 4.7987
- 社交媒体广告效果显著：系数 1.2028
- TV 广告效果较弱：系数 1.5102
- 节假日效应弱于北美：系数 18.2465

**结论**：欧洲市场广播广告效果最好，社交媒体广告有效，电视广告效果一般，节假日对销量的拉动弱于北美。

## 三、截距项处理方式说明
### 1. 具体处理方式
截距项采用**在数据预处理阶段手动添加一列全 1 列**的方式实现：
X = np.hstack([np.ones((len(X), 1)), X])
不在模型内部自动处理
###2. 对结果的影响

    1.严格符合经典 OLS 数学公式
    系数估计式 β̂ = (XᵀX)⁻¹Xᵀy 要求设计矩阵包含截距，手动添加保证数学推导与计算完全一致。
    2.避免模型强制过原点
    不添加截距会使模型被迫从 0 点开始拟合，导致基础销量无法被正确估计，模型偏差大幅上升。
    3.F 检验与统计推断更准确
    截距项参与自由度计算与协方差矩阵构造，手动添加保证了假设检验（F 检验）的正确性。
    4.与 sklearn 对比公平可控
    可以关闭 sklearn 自动截距项，使自定义模型与工业库模型在完全相同的设定下对比精度与速度。
