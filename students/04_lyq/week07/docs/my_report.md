# 实验展示重点说明

## 一、GradientDescentOLS 实现方式
1. **模型结构**
   - 初始化系数为 0，记录每一轮 loss 到 `loss_history_`。
   - 支持两种梯度下降模式：`full_batch` 和 `mini_batch`。

2. **梯度计算**
   - 损失函数：MSE
   - 梯度公式：
     \[
     grad = \frac{2}{N} X^T (y_{pred} - y_{true})
     \]
   - 系数更新：
     \[
     coef = coef - lr * grad
     \]
   - 模型维护属性：

    coef_：回归系数
    loss_history_：每轮迭代的 loss，用于绘制学习曲线

3. **模式实现**
   - **full_batch**：每一轮使用**全部训练样本**计算梯度。
   - **mini_batch**：每一轮随机抽取**20% 样本**计算梯度。

4. **收敛与停止**
   - 连续两次 loss 变化小于 `tol=1e-5` 则**提前停止**。
   - 最大迭代轮数 `max_iter` 限制。

5. **截距项处理**
   - 在**特征矩阵最左侧手动添加一列 1**。
   - 截距项**不参与标准化**，只参与梯度下降更新。

---

## 二、full_batch 与 mini_batch loss 曲线差异
1. **Full Batch GD**
   - 曲线**平滑、稳定、单调下降**。
   - 每一步都使用全部数据，方向准确，无波动。
   - 对学习率非常敏感，**学习率偏大容易发散（loss 爆炸）**。

2. **Mini Batch GD**
   - 曲线**有轻微抖动、波动**，但整体快速下降。
   - 因为每次只用部分样本，梯度带噪声，所以不光滑。
   - 收敛速度通常更快，对学习率更宽容。

3. **直观总结**
   - Full Batch：平稳、慢、敏感、易发散。
   - Mini Batch：波动、快、稳定、不易发散。

## 三、最佳学习率选择
LR=0.1      | Val R2=0.9029 | Val RMSE=71.5727
LR=0.01     | Val R2=0.9026 | Val RMSE=71.6818
LR=0.001    | Val R2=0.5986 | Val RMSE=145.5163
LR=0.0001   | Val R2=-9.0006 | Val RMSE=726.2934
LR=1e-05    | Val R2=-13.2044 | Val RMSE=865.5861

通过验证集 R² 从 `[0.1, 0.01, 0.001, 0.0001, 1e-5]` 中选择：
**最佳学习率 = 0.1**

- 0.1 时验证集 R² 最高。
- 0.01 表现接近但略低。
- 0.001 及以下收敛过慢，效果明显变差。
- 1e-5、1e-4 模型无法有效学习，R² 为负。

## 四、为什么不能在全部数据上先做标准化（数据泄露）
1. **如果先对全部数据（train+val+test）标准化**
   - 模型训练时**间接使用了验证集与测试集信息**。
   - 评估结果虚高，**无法代表真实泛化能力**。
   - 这是机器学习中的典型错误。

2. **正确做法（我严格实现）**
   - 只用训练集拟合 `scaler`，得到均值和标准差。
   - 验证集、测试集**只使用 transform**，不重新计算。
   - 保证数据不会从测试集泄露到训练过程。

3. **结论**
   正确标准化是**模型评估可信的基础**。

## 五、Test 集结果：GradientDescentOLS 与 AnalyticalOLS 非常接近
### 测试集结果
- GD：R²=0.8913，RMSE=78.4113
- OLS：R²=0.8906，RMSE=78.6485

### 为什么几乎一样
1. **两者都是线性回归模型**
   - 目标都是最小化 MSE。
   - 理论最优解相同。

2. **GradientDescent 已收敛到最优解附近**
   - 使用最佳学习率 0.1。
   - 足够迭代次数 + 早停。
   - 梯度下降足够逼近解析解。

3. **数据线性度较好**
   - 特征与目标线性关系强，无复杂非线性。
   - 线性模型本身就能很好拟合。

### 结论
梯度下降正确实现，**收敛到全局最优**，因此与解析解结果几乎一致。

## 六、发散现象分析
部分学习率下发散（R² 为负，loss 极大）原因：
- **学习率太大**：梯度下降迈大步，**越过最小值，loss 不断上升**。
- **Full Batch 尤其敏感**：因为每一步方向太准，步子大直接发散。
- **学习率太小**：下降太慢，迭代不足，无法收敛。

本次实验中：
- lr=0.1、0.01 正常收敛
- lr ≤ 0.001 收敛过慢
- lr 太小导致模型几乎没学到东西
